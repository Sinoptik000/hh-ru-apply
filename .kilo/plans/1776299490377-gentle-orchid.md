# План: Кнопка массового обновления незагруженных вакансий

## Контекст задачи

На карточке вакансии уже есть значок обновления, который:
- Делает POST-запрос на `/api/vacancy/refresh-body`
- Обновляет текст вакансии с hh.ru
- Определяет "незагруженные" данные по критерию: `description.length < 80`

**Задача:** Добавить аналогичную кнопку рядом с полем поиска, которая будет обновлять данные ВО ВСЕХ незагруженных вакансиях текущего раздела (На проверке, Подходят, Отклонённые).

---

## Архитектурные решения

### 1. UI компонент — кнопка рядом с поиском

**Расположение:** `dashboard/public/index.html` — блок `.search-panel` или `.controls`

**Визуально:**
- Иконка 🔄 (аналог той что на карточке)
- Счётчик рядом: "12" — количество вакансий требующих обновления
- Состояния:
  - **Default:** серая/синяя иконка
  - **Hover:** подсветка
  - **Active (идёт обновление):** вращающаяся иконка + счётчик "5/12"
  - **Disabled:** когда нет вакансий требующих обновления — скрыть или сделать неактивной

### 2. Определение "незагруженных" вакансий

Критерий оставить существующий из `lib/refresh-vacancy-from-hh.mjs`:
```javascript
function looksParsedOk(parsed) {
  const d = String(parsed?.description || '').trim();
  return d.length >= 80;
}
```

На фронтенде проверить:
```javascript
const needsRefresh = (item) => {
  const desc = item.descriptionPreview || '';
  return desc.trim().length < 80;
};
```

### 3. Логика批量ного обновления

**Подход:** Последовательное обновление (не并行) — из-за нагрузки на hh.ru и возможных rate limits.

**Лимит:** Максимум 20 вакансий за один клик. Если незагруженных больше — показать кнопку "Продолжить" после завершения 20.

```javascript
const BATCH_LIMIT = 20;

async function refreshIncompleteInSection() {
  const incomplete = currentItems.filter(needsRefresh);
  const batch = incomplete.slice(0, BATCH_LIMIT);
  const hasMore = incomplete.length > BATCH_LIMIT;

  let completed = 0;

  for (const item of batch) {
    await refreshSingleVacancy(item.id);
    completed++;
    updateCounter(completed, batch.length);
  }

  if (hasMore) {
    showContinueButton(); // Показать кнопку "Продолжить (осталось N)"
  }
}
```

### 4. Структура данных для отслеживания прогресса

**Клиентское состояние:**
```javascript
let bulkRefreshState = {
  active: false,
  total: 0,
  completed: 0,
  currentId: null
};
```

**UI во время обновления:**
- Кнопка показывает вращающуюся иконку
- Счётчик: "5/12" (обновлено/всего)
- По завершении: toast "Обновлено 12 вакансий"

### 5. Обработка ошибок

- При ошибке одной вакансии — продолжить со следующей
- По завершении показать отчёт: "Обновлено 10 из 12. 2 ошибки"
- Логировать ошибки в консоль для отладки

### Шаг 3: Добавить функцию определения незагруженных в `app.js`

```javascript
function needsVacancyBodyRefresh(item) {
  const desc = item.descriptionPreview || '';
  return desc.trim().length < 80;
}

function getIncompleteCount(items) {
  return items.filter(needsVacancyBodyRefresh).length;
}
```

### Шаг 4: Добавить функцию批量ного обновления в `app.js`

```javascript
async function refreshIncompleteInSection() {
  const incomplete = currentItems.filter(needsVacancyBodyRefresh);
  if (incomplete.length === 0) return;

  setRefreshSectionLoading(true, incomplete.length);

  let success = 0;
  let failed = 0;

  for (const item of incomplete) {
    try {
      const res = await api('/api/vacancy/refresh-body', {
        method: 'POST',
        body: JSON.stringify({ id: item.id })
      });
      if (res.ok) success++;
      else failed++;
    } catch (e) {
      failed++;
      console.error(`Failed to refresh vacancy ${item.id}:`, e);
    }

    updateRefreshSectionProgress(success + failed, incomplete.length);
  }

  invalidateCache();
  await load();
  setRefreshSectionLoading(false);
  showToast(`Обновлено ${success} из ${incomplete.length}${failed ? ` (${failed} ошибок)` : ''}`, failed ? 'neutral' : 'good');
}
```

### Шаг 5: Привязать обработчик и обновление счётчика

- При переключении табов (смене раздела) — пересчитать количество незагруженных
- При клике на кнопку — запустить批量ное обновление
- При завершении — обновить UI

## Поведение

| Ситуация | Поведение |
|----------|-----------|
| В разделе все вакансии загружены | Кнопка скрыта или неактивна |
| Клик по кнопке | Последовательное обновление (до 20 штук за раз) |
| Незагруженных больше 20 | После 20 показать кнопку "Продолжить" |
| Во время обновления | Вращающаяся иконка + прогресс "3/12" |
| Ошибка при обновлении одной | Продолжить со следующей, показать итоговую статистику |
| Завершение | Toast с результатом, обновить список |
