# План: Исправить кнопку "Добавить" — убрать конфликт с Chrome

## Проблема
`add-vacancy-worker.mjs` запускает Chromium с профилем `data/session/chromium-profile`. Этот профиль уже используется обычным Chrome пользователя (hh.ru открыт в браузере). Playwright не может открыть браузер с занятым профилем → ошибка "Target page, context or browser has been closed".

## Решение
**Использовать отдельный изолированный профиль для автоматизации**, который не конфликтует с обычным Chrome пользователя.

### Шаг 1: Изменить путь профиля в `add-vacancy-worker.mjs`
Вместо `data/session/chromium-profile` использовать `data/session/chromium-automation-profile`.

Нужно изменить `scripts/add-vacancy-worker.mjs`:
- Убрать проверку `sessionProfilePath()` (там используется профиль пользователя)
- Использовать новый путь `path.join(DATA_DIR, 'chromium-automation-profile')` — это будет ИЗОЛИРОВАННЫЙ профиль, который не конфликтует с обычным Chrome

### Шаг 2: Пользователь выполняет вход один раз в автоматизации
После изменения пути профиля, при первом запуске `npm run login` нужно:
- Либо сохранять сессию в новый путь
- Либо пользователь один раз запускает `HH_AUTOMATION_PROFILE=1 node scripts/login.mjs` для входа в hh.ru с новым профилем

### Шаг 3: Убедиться что `login.mjs` тоже использует отдельный профиль
Проверить `scripts/login.mjs` — он тоже может использовать `sessionProfilePath()` и его тоже нужно перевести на новый изолированный профиль.

## Что нужно сделать пользователю после реализации плана:
1. Один раз выполнить `npm run login` (или специальную команду) — это войдёт в аккаунт hh.ru в ИЗОЛИРОВАННОМ профиле автоматизации
2. После этого кнопка "Добавить" будет работать даже когда обычный Chrome открыт

## Примечание
- Обычный Chrome пользователя продолжает работать как обычно
- Автоматизация использует СВОЙ отдельный профиль без конфликтов
- Это стандартная практика — не использовать один профиль для ручного браузера и для автоматизации
