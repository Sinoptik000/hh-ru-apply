# План: Поле поиска вакансий на дашборде

## Задача
Добавить поле поиска на страницу дашборда для фильтрации вакансий по тексту внутри текущего раздела (tab).

## Направление поиска
Поиск ведётся по следующим полям вакансии:
- `title` — название вакансии
- `company` — название компании
- `geminiTags` — теги, присвоенные AI
- `searchQuery` — ключевое слово, которым была найдена вакансия

Поиск **регистронезависимый**, совпадение — если **любое** из полей содержит искомую строку.

#### 3.2. Обработчик ввода в поле поиска
Добавить функцию-обработчик `onSearchInput`, которая:
1. Получает значение из `#vacancy-search`

```javascript
function onSearchInput(e) {
  const query = e.target.value.trim();
  currentSearchQuery = query;
  const clearBtn = document.getElementById('vacancy-search-clear');
  if (clearBtn) clearBtn.style.display = query ? '' : 'none';
  renderVacancyList();
}
```

#### 3.3. Обработчик очистки
Добавить функцию `clearSearch`, которая:
1. Очищает `currentSearchQuery`

```javascript
function clearSearch() {
  currentSearchQuery = '';
  const input = document.getElementById('vacancy-search');
  if (input) input.value = '';
  const clearBtn = document.getElementById('vacancy-search-clear');
  if (clearBtn) clearBtn.style.display = 'none';
  renderVacancyList();
}
```

#### 3.4. Фильтрация в `renderVacancyList()`

```javascript
if (currentSearchQuery) {
  const q = currentSearchQuery.toLowerCase();
  vacancies = vacancies.filter(v => {
    const titleMatch = (v.title || '').toLowerCase().includes(q);
    const companyMatch = (v.company || '').toLowerCase().includes(q);
    const tagsMatch = (v.geminiTags || []).some(t => t.toLowerCase().includes(q));
    const queryMatch = (v.searchQuery || '').toLowerCase().includes(q);
    return titleMatch || companyMatch || tagsMatch || queryMatch;
  });
}
```

## Поведение

1. **Ввод текста** → список фильтруется мгновенно (без задержки/debounce)
2. **Очистка** → кнопка × появляется только при наличии текста, по клику сбрасывает поиск
3. **Смена таба** → поисковый запрос **сохраняется** при переключении между табами (фильтрация работает независимо в каждом разделе)
4. **Пустой результат** → показать сообщение "Ничего не найдено" внутри списка
