# Подбор названий вакансий — методология

## Проблема

Названия вакансий на hh.ru часто не совпадают с реальным содержанием работы:
- "Операционный менеджер" в одной компании = координация процессов
- "Операционный менеджер" в другой = работа в офисе на ресепшене
- "Project Manager" в IT = управление разработкой
- "Project Manager" в строительстве = управление стройкой

## Метод анализа

### 1. Декомпозиция опыта кандидата

Выдели **конкретные действия** из резюме, а не просто должности:

```
Что делал:
- Координировал работу отделов (продажи, клиентский сервис, операции)
- Выстраивал бизнес-процессы с нуля
- Настраивал CRM и автоматизацию
- Управлял командами до 30-50 человек
- Запускал филиалы/продукты с нуля
- Разрабатывал регламенты и базы знаний
- Контролировал исполнение задач и сроки
- Готовил отчеты для руководства
```

### 2. Исключение нерелевантных ролей

| Исключить | Почему |
|-----------|--------|
| Project Manager IT | Управление разработкой софта, а не бизнес-процессами |
| Account Manager | Просто ведение текущих клиентов, без операционки |
| Chief of Staff (HR) | HR-директор, не про операционное управление |
| Офис-менеджер | Административная поддержка, не управление процессами |
| Sales Manager | Только продажи, без координации других отделов |
| Product Manager | Управление продуктом, не операционкой компании |

### 3. Поиск по функциям, а не по названиям

Ключевые функции операционного менеджера:
- Координация работы отделов
- Контроль исполнения задач
- Построение процессов и регламентов
- Отчетность для руководства
- Автоматизация бизнес-процессов
- Управление операционной деятельностью

### 4. Релевантные названия вакансий

**Прямые совпадения:**
- Операционный менеджер
- Менеджер по операционной деятельности
- Координатор операционной деятельности
- Operations manager
- Бизнес-координатор

**Смежные роли (операционка + управление):**
- Руководитель операционного отдела
- Руководитель направления
- Менеджер по управлению бизнес-процессами
- Менеджер по организационному развитию
- Менеджер по автоматизации процессов

**Специфика стартапов:**
- Операционный менеджер стартапа
- Руководитель операционного блока
- Менеджер по развитию бизнеса (если есть операционка)
- Руководитель проекта (если про процессы, не про разработку)

## Правила составления search-keywords.txt

1. **Приоритет точных совпадений** — "операционный менеджер", "координатор"
2. **Функциональные запросы** — "управление бизнес-процессами", "автоматизация процессов"
3. **Исключить IT-разработку** — не добавлять "project manager IT", "scrum master"
4. **Исключить чистые продажи** — не добавлять "sales manager", "account manager" без контекста операционки
5. **Добавлять только то, что кандидат реально делал** — если запускал филиалы, то "руководитель филиала"

## Пример правильного списка

```
Операционный менеджер
Координатор операционной деятельности
Менеджер по операционной деятельности
Operations manager
Руководитель операционного отдела
Менеджер по управлению бизнес-процессами
Бизнес-координатор
Менеджер по организационному развитию
Менеджер по автоматизации процессов
Руководитель направления
Менеджер по развитию бизнеса с операционными функциями
Руководитель проекта (не IT-разработка)
Менеджер по внедрению CRM
Менеджер по оптимизации бизнес-процессов
Бизнес-аналитик (процессы, не данные)
```

## Как проверить релевантность вакансии

Перед добавлением в очередь проверь:

- [ ] В описании есть про координацию/управление процессами?
- [ ] Требуется опыт в операционном управлении?
- [ ] Это НЕ управление разработкой софта?
- [ ] Это НЕ чистые продажи без операционки?
- [ ] Кандидат реально мог бы выполнять эти задачи?

Если хоть один пункт "нет" — исключаем.
