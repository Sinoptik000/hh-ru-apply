# Сопроводительные письма — стиль и ATS-оптимизация

## Архитектура генерации

| Файл | Роль |
|------|------|
| `config/cover-letter.txt` | Основной шаблон — структура, блоки, инструкции для LLM |
| `config/cover-letter-style-examples.txt` | Эталоны стиля — LLM имитирует ритм, тон, длину фраз |
| `CV/` | Резюме кандидата — факты для привязки к требованиям вакансии |
| `lib/cover-letter-openrouter.mjs` | Промпт + вызов OpenRouter для генерации 3 вариантов |
| `lib/cover-letter-style-context.mjs` | Загрузка контекста стиля (примеры + правки пользователя) |

LLM получает: шаблон + стиль-примеры + резюме + описание вакансии → генерирует 3 варианта.

## Структура сопроводительного письма

Письмо состоит из блоков, разделённых **пустыми строками**:

```

Меня заинтересовала работа в вашей компании.

Вакансия: [название вакансии].

[Input: отзеркаливание + ATS-список]

Обо мне:
* ...

Ключевые достижения:
* ...

Компетенции:
* ...

Навыки работы с ИИ / Ai:
* ...

Владею инструментами:
* ...

Опыт:
* ...

Спасибо за уделенное время!

Мои контакты:

Эдуард Фридман
Telegram: https://t.me/EduardFri

## Правило 1: Отзеркаливание вакансии (блок Input)

**Цель:** рекрутер мгновенно узнаёт свои формулировки — понимает, что кандидат внимательно прочитал вакансию.

**Как работает:**

1. LLM анализирует описание вакансии и выделяет суть: какую роль ищут и для каких задач.
2. Формирует 1-2 предложения по шаблону:
   > «Вы ищете [роль], который будет [ключевые задачи] и обеспечит [требования/результат]».
3. Использует формулировки **максимально близко к тексту вакансии** — те же слова, тот же порядок.

**Пример:**

> Вы ищете операционного менеджера, который поможет руководителю держать компанию в порядке, в сроках и в управленческом ритме - выстроит систему, будет координировать работу отделов, контролировать исполнение задач и сроков, разрабатывать регламенты.

**Почему это работает:**

- Рекрутер видит **свои слова** — это создаёт эффект «своего человека»
- ATS получает **точные совпадения** ключевых фраз
- Нет ощущения шаблонной рассылки

## Правило 2: Прохождение ATS-фильтров

**Что такое ATS:** система автоматического скрининга резюме, которая ищет **точные совпадения** ключевых слов из требований вакансии.

### ATS-список в письме

Сразу после блока Input вставляется маркированный список из **3-5 ключевых требований** вакансии, привязанных к опыту кандидата.

**Формат каждой строки:**
```
* [требование из вакансии] - [мой релевантный опыт/результат]
```

**Пример:**
```
* Координация работы отделов и контроль исполнения задач - управлял кросс-функциональными командами до 30 человек, доводил задачи до результата
* Разработка регламентов и стандартов - создавал базы знаний, регламенты и прайсы в Wata Payments и «Огонь»
* Подготовка отчетности и работа с данными - готовил сводки, аналитические отчеты для руководства, уверенно работал с Excel и Google Sheets
* Внедрение CRM и таск-менеджеров - настраивал AmoCRM, Bitrix24, Pipedrive, автоматизировал сбор лидов и воронки
* Выявление слабых мест в процессах - проводил конкурентный анализ, оптимизировал бизнес-процессы (BPMN), сокращал время квалификации лидов на 30%
```

### Правила ATS-оптимизации

| Правило | Почему |
|---------|--------|
| Используй **те же термины** что в вакансии | ATS ищет точные совпадения — «BPMN» ≠ «бизнес-процессы» |
| Не заменяй синонимами | «AmoCRM» ≠ «CRM-система» |
| Привязывай к конкретному опыту | Без фактов выглядит как пустой список |
| 3-5 пунктов | Больше — перегруз; меньше — мало для ATS |
| Не пиши «для прохождения ATS» | Рекрутер не должен видеть механику |

## Правило 3: Избегание признаков ИИ-текста

LLM склонен к шаблонным паттернам, которые выдают машинную генерацию. Вот что **категорически запрещено**:

### Запрещённые паттерны

| Признак ИИ | Как писать вместо этого |
|------------|------------------------|
| Длинное тире «—» | Короткий дефис «-» |
| «Уважаемые рекрутеры / меня зовут / я пишу вам, чтобы...» | Сразу к делу: «Меня заинтересовала...» |
| Цепочки прилагательных: «глубокие знания», «уникальный опыт», «идеально подхожу» | Конкретные факты: «сократил на 30%», «управлял 30 человек» |
| Общие фразы: «динамично развивающаяся компания», «лидер отрасли» | Конкретика из вакансии |
| Markdown-разметка (`**жирный**`, `# заголовок`) | Простой текст с `*` для списков |
| Код-блоки (```json) в ответе | Чистый текст |
| «Я обладаю глубокими компетенциями» | «Умею превращать хаос в работающие системы» |
| Идеально сбалансированные предложения | Разная длина — как у живого человека |

### Дополнительные признаки ИИ (чего избегать)

| Признак | Описание |
|---------|----------|
| **Точная симметрия абзацев** | Все абзацы одной длины — неестественно |
| **Отсутствие «я» или избыток «я»** | Одно-два уместных «я» — нормально |
| **Идеальная пунктуация** | Живой текст допускает короткие рубленые фразы |
| **«Буду рад возможности»** | Шаблонная концовка — лучше «Готов созвониться в удобное время» |
| **«В заключение хочу отметить»** | ИИ любит формальные переходы — убирай |
| **Слишком много восторга** | «Впечатляет», «восхищаюсь» — не для делового письма |

### Анти-ИИ правила в промпте

```
Жёстко избегай признаков «нейросетевого» текста:
- не начинай с «Уважаемые рекрутеры/меня зовут/я пишу вам, чтобы…» шаблонно;
- не используй цепочки прилагательных и пустые усилители;
- не перечисляй качества списком без привязки к фактам из резюме;
- допускай разговорные короткие фразы, одно уместное «я»;
- конкретика из вакансии и CV, не общие слова.
```

## Правило 4: Пустые строки между блоками

Каждый смысловой блок отделяется **пустой строкой**. Это не просто форматирование — это ритм письма.

**Обязательные отступы:**

| Место | Правило |
|-------|---------|
| После `Здравствуйте!` | ВСЕГДА пустая строка |
| После `Меня заинтересовала...` | Пустая строка |
| После названия вакансии | Пустая строка |
| После Input-абзаца | Пустая строка |
| После ATS-списка | Пустая строка |
| Между всеми секциями (Обо мне, Достижения, и т.д.) | Пустая строка |
| Между `Спасибо за уделенное время!` и `Мои контакты:` | ВСЕГДА пустая строка |

## Чек-лист проверки письма

Перед отправкой проверь:

- [ ] Нет длинных тире «—» — только дефисы «-»
- [ ] Есть отзеркаливание вакансии (1-2 предложения)
- [ ] Есть ATS-список (3-5 пунктов с точными терминами из вакансии)
- [ ] Нет маркеров ИИ-текста (см. таблицу запрещённых паттернов)
- [ ] Пустые строки между всеми блоками
- [ ] Контакты в конце (Telegram, Email)
- [ ] Нет markdown-разметки (`**`, `#`, ``` ```)
- [ ] Тон — деловой, но не канцелярский
- [ ] Конкретика вместо общих фраз
