# Сопроводительные письма на вакансии

## Цель:
составить отклик на вакансию в виде сопроводительного письма, которое:
  - соответствует требованиям вакансии
  - соответствует моему**шаблону письма**.
  - оптимизировано для прохождения ATS-фильтров

Отклик на вакансию составляется на основании этих источников:
- **Описание вакансии**
- **Этот skill**
- **Шаблоны писем** для конкретной профессии (например, `cover-letter-ops.txt` или `cover-letter-sales.txt`).
- ATS оптимизации: ключевых слов и выражений, которые нужно вставить в письмо, чтобы пройти автоматический отбор.

## Тональность текста

- Пиши как кандидат, а не HR-специалист
- Деловой стиль, без канцелярита
- Избегай клише: "глубокие знания", "готов приложить максимум"
- Короткие фразы с живым стилем

Если в вакансии указано имя рекрутера, обращайся по имени: «Здравствуйте, [Имя]!». 
Если имя не указано, используй нейтральное приветствие: «Здравствуйте!».

## Блок `Отзеркаливание вакансии`

**Цели:** 
повторить формулировки, указанные в вакансии, чтобы:
- рекрутер увидел знакомые формулировки и счел что кандидат внимательно прочитал вакансию. 
- пройти ATS-фильтры, которые ищут точные совпадения ключевых слов.

**Как работает:**

1. LLM анализирует описание вакансии и требования к кандидату.
2. Затем выделяет суть: какую роль ищут и для каких задач.
3. Формирует Отзеркаливание по шаблону:
   > «Вы ищете [роль], который будет выполнять [ключевые задачи] и обеспечит компании [требования/результат]».
3. Используй формулировки **максимально близко к тексту вакансии** — те же слова, тот же порядок.

**ATS-оптимизация** - обязательно примени. Вставь 1-2 ключевых слова из вакансии, чтобы пройти фильтры. Но не перегружай текст — 3-5 ключевых слов достаточно.

## Блок `Навыки`

- этот блок надо органично/нативно заполнить на основе **требований вакансии** и моего шаблона. 
- Приоритет:  **требованиям вакансии**. 
- Следи чтобы не было дублирования формулировок из шаблона.
- используй те же термины что в вакансии, т.к. ATS ищет точные совпадения («BPMN» ≠ «бизнес-процессы»), не заменяй синонимами — «AmoCRM» ≠ «CRM-система»).

**ATS-оптимизация**: обязательно примени.

## Блок `Владею инструментами`

- этот блок надо органично/нативно заполнить на основе **требований вакансии** и моего шаблона. 
- Приоритет:  **шаблон письма**. добавляй инструменты из вакансии, которых нет в шаблоне, но не удаляй инструменты из шаблона, которых нет в вакансии.

**ATS-оптимизация**: обязательно примени.

## Правило: Прохождение ATS-фильтров

**ATS-фильтры** — системы, которые отклоняют до 75% откликов без ключевых слов из вакансии.

**Цель:** органично вставить 3-5 ключевых слов в блоки "Отзеркаливание", "Навыки", "Владею инструментами".

**Правила:**
- Используй точные термины из вакансии («BPMN» ≠ «бизнес-процессы»)
- Не заменяй синонимами («AmoCRM» ≠ «CRM-система»)
- Не перегружай текст — 3-5 слов достаточно
- Не применяй в блоке "Обо мне"

## Правило: Избегание признаков ИИ-текста

LLM склонен к шаблонным паттернам. Категорически запрещено:

### Запрещённые паттерны

- Запрещено использовать длинное тире «—» → используй короткий дефис «-»
- «Уважаемые рекрутеры / меня зовут / я пишу вам, чтобы…» → сразу к делу: «Меня заинтересовала…»
- Цепочки прилагательных: «глубокие знания», «уникальный опыт», «идеально подхожу» → конкретные факты: «сократил на 30%», «управлял 30 человек»
- Общие фразы: «динамично развивающаяся компания», «лидер отрасли» → конкретика из вакансии
- Markdown-разметка (`**жирный**`, `# заголовок`) → простой текст с `*` для списков
- Код-блоки (```json) в ответе → чистый текст
- «Я обладаю глубокими компетенциями» → «Умею превращать хаос в работающие системы»
- Идеально сбалансированные предложения → разная длина, как у живого человека

### Дополнительные признаки ИИ (чего избегать)

- Точная симметрия абзацев — все абзацы одной длины неестественно
- Отсутствие «я» или избыток «я» — одно-два уместных «я» нормально
- Идеальная пунктуация — живой текст допускает короткие рубленые фразы
- «Буду рад возможности» — шаблонная концовка, лучше «Готов созвониться в удобное время»
- «В заключение хочу отметить» — ИИ любит формальные переходы, убирай
- Слишком много восторга — «Впечатляет», «восхищаюсь» не для делового письма
