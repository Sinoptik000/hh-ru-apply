# Workflows

## Полный цикл работы с hh-ru-apply

### 1. Первый запуск
1. `npm install` - установить зависимости
2. `npx playwright install chromium` - установить браузер
3. `cp .env.example .env` - скопировать переменные окружения
4. `npm run login` - войти в hh.ru аккаунт

### 2. Настройка проекта
1. Создать `config/search-keywords.txt` с поисковыми запросами
2. Добавить резюме в папку `CV/`
3. Настроить `config/cover-letter.txt` - шаблон сопроводительного письма
4. Опционально: добавить `config/cover-letter-style-examples.txt` с примерами стиля
5. Настроить OpenRouter API ключ в `config/secrets.local.env`

### 3. Рабочий цикл
1. `npm run vacancies` - собрать вакансии
2. `npm run harvest` - оценить вакансии через LLM
3. `npm run dashboard` - запустить веб-интерфейс
4. В дашборде: просмотреть вакансии, сгенерировать письма, утвердить
5. Откликнуться: через дашборд или `npm run hh-apply-chat -- --id=<uuid>`

## Диагностика проблем

### Шаг 1: Проверка состояния
- Проверить логи последнего запуска
- Проверить содержимое `data/vacancies-queue.json`
- Проверить статус сессии в `data/session/`

### Шаг 2: Поиск причины
- Если "element not found" - обновить селекторы через `npm run codegen-hh`
- Если авторизация - запустить `npm run login`
- Если LLM не работает - проверить API ключ и модель

### Шаг 3: Исправление
- Исправить найденную проблему
- Перезапустить соответствующий скрипт
- Проверить результат

## Работа с JobCRM (предполагаемая интеграция)

### Подключение к CRM
1. Настроить API ключи в `config/secrets.local.env`
2. Создать конфиг подключения в `config/crm-config.json`
3. Протестировать подключение через тестовый скрипт

### Синхронизация данных
1. Экспортировать вакансии из CRM
2. Сопоставить с данными hh.ru
3. Обновить статусы откликов
4. Импортировать новые кандидатуры

### Автоматизация отчетов
1. Собрать статистику по откликам
2. Сгенерировать отчет в требуемом формате
3. Отправить в CRM через API

## Деплой и обновления

### Локальный деплой
1. `git pull` - получить обновления
2. `npm install` - обновить зависимости
3. `npm run dashboard` - запустить для проверки

### Docker деплой
1. `docker compose pull` - получить новый образ
2. `docker compose up -d` - перезапустить сервисы
3. Проверить логи через `docker compose logs`

### Обновление селекторов
1. `npm run codegen-hh` - авто-генерация
2. Или вручную через `npx playwright codegen https://hh.ru`
3. Обновить файлы `lib/hh-response-selectors.mjs` и `lib/hh-chat-selectors.mjs`
