# Rules

## Глобальные запреты

### Файлы
- Не создавать лишние файлы без необходимости
- Все новые файлы должны быть обоснованы
- Не использовать markdown-разметку в сопроводительных письмах (**, #, ```)
- Избегать длинных тире «—» в письмах, использовать дефис «-»

### ИИ-текст в письмах
- Категорически запрещены признаки ИИ-текста:
  - Цепочки прилагательных: "глубокие знания", "уникальный опыт"
  - Общие фразы: "динамично развивающаяся компания"
  - Шаблонные начала: "Уважаемые рекрутеры", "меня зовут"
  - Идеально сбалансированные предложения
  - Отсутствие "я" или избыток "я"
  - "Буду рад возможности" - использовать конкретные формулировки

### Безопасность
- Никогда не коммитить секретные ключи или пароли
- API ключи хранить только в config/secrets.local.env
- Не логировать чувствительные данные

## Стандарты

### Диагностика
- Всегда использовать step-by-step подход при решении проблем
- Проверять логи перед предположениями
- Тестировать изменения поэтапно

### Код
- Следовать существующим паттернам проекта
- Использовать async/await для асинхронных операций
- Добавлять комментарии только при необходимости
- Соблюдать принципы DRY и KISS

### Коммуникация
- Краткие ответы без лишней вежливости
- Фокус на технических деталях
- Прямые инструкции

### Проектная структура
- CV/ - для резюме (не в git)
- config/ - для конфигурационных файлов
- lib/ - для библиотечного кода
- data/ - для собранных данных
- docs/ - для документации
