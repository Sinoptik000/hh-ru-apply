# AGENTS-First — Центральное хранилище инструкций

## О проекте

**hh-ru-apply** — автоматизированная система откликов на вакансии hh.ru.

Возможности: сбор вакансий, AI-оценка через OpenRouter, генерация сопроводительных писем, автоматический отклик через браузер (Playwright).

## Архитектура

Этот проект использует принцип **"The Folder is the Agent"**. Все инструкции для ИИ-агентов хранятся в папке `AGENTS/`.

## Структура AGENTS/

| Файл | Назначение |
|------|------------|
| `AGENTS/identity.md` | Кто я (Эдуард), стиль общения, мой стек |
| `AGENTS/skills.md` | Инструменты проекта и /-команды Kilo CLI |
| `AGENTS/rules.md` | Глобальные запреты и стандарты |
| `AGENTS/workflows.md` | Пошаговые алгоритмы (деплой, фикс багов, работа с hh-ru-apply) |
| `AGENTS/logs/` | Временные логи работы |

## Для новых агентов

При старте работы с этим проектом:

1. Прочитай `AGENTS/identity.md` — кто такой Эдуард
2. Прочитай `AGENTS/rules.md` — что запрещено
3. Прочитай `AGENTS/skills.md` — какие инструменты доступны
4. Прочитай `AGENTS/workflows.md` — как решать типичные задачи

## Маяки IDE

В корне проекта есть файлы-маркеры для различных IDE:
- `.qwenrules` — Qwen
- `.cursorrules` — Cursor
- `.windsurfrules` — Windsurf
- `GEMINI.md` — Gemini
- `.clinerules` — Cline

Все они указывают на папку `AGENTS/` как центральный источник инструкций.

## Настройки Qwen

Локальные настройки Qwen: `.qwen/settings.json` (permissions для текущей машины).

Навыки Qwen интегрированы в `AGENTS/skills.md`:
- `cover-letter-ats-style` — ATS-оптимизация писем
- `hh-boolean-search` — поисковые операторы hh.ru
- `vacancy-title-matcher` — подбор названий вакансий

## Правило самоподдержания

Если нужно добавить новый навык или изменить правило — редактируй файл в `AGENTS/`. Если создаёшь нового агента или микросервис — он должен начинать работу с чтения папки `AGENTS/`.
