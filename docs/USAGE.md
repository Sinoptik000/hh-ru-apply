## Установка

### 1. Клонируйте репозиторий

```bash
git clone <ваш-repo-url>
cd hh-ru-apply
```

### 2. Установите зависимости

```bash
npm install
```

### 3. Установите браузер Playwright

```bash
npx playwright install chromium
```

Скачается Chromium, который будет использоваться для автоматизации.

### 4. Скопируйте переменные окружения

```bash
cp .env.example .env
```

Теперь редактируйте `.env` по мере необходимости (см. разделы ниже).

## Первый запуск и вход

```bash
npm run login
```

**Что произойдёт:**

1. Откроется окно Chromium
2. Перейдёт на hh.ru
3. Вам нужно **войти в свой аккаунт** — ввести логин, пароль, пройти капчу и двухфакторную аутентификацию, если она включена
4. После успешного входа вернитесь в терминал и нажмите **Enter**

**Важно:**

- Браузерный профиль сохраняется в `data/session/chromium-profile`
- При последующих запусках логиниться **не нужно** — скрипты переиспользуют тот же профиль
- Сессия со временем «протухает» — если скрипт ругается на авторизацию, запустите `npm run login` снова

## Настройка поиска

Откройте `config/search-keywords.txt` и напишите поисковые запросы, по одному на строку:

```
python backend разработчик
senior python developer москва удалённо
python data engineer
django backend
```

**Советы:**

- Чем конкретнее запрос, тем релевантнее результат
- Добавьте город и тип работы, если важно
- Не задавайте слишком общие запросы вроде просто `python` — будет много мусора

Настройки лимитов и пауз находятся в `.env`:

```bash
HH_PER_KEYWORD_LIMIT=8        # сколько вакансий брать по каждому ключу
HH_MAX_TOTAL=7                # максимум вакансий за запуск
HH_OPEN_DELAY_MIN_MS=3000     # минимальная пауза между открытиями
HH_OPEN_DELAY_MAX_MS=5000     # максимальная пауза между открытиями
```

Увеличивая `HH_MAX_TOTAL`, можно собрать больше вакансий за один запуск.

## Добавление резюме

Создайте папку `CV/` в корне проекта и положите туда ваши резюме (.md, .txt или .pdf).

**Формат `.md` (рекомендуется):**

```markdown
# Имя Фамилия

## Опыт
Компания: Senior Python Developer
- Разработка микросервисов...
- ...

## Навыки
Python, FastAPI, PostgreSQL, docker, kubernetes
```

Файлы из `CV/` **не попадают** в git — папка добавлена в `.gitignore`.

Эти резюме используются скриптом `harvest` при оценке — LLM сравнивает текст резюме с требованиями вакансии и даёт скор совпадения.

## Настройка AI для оценки вакансий

### OpenRouter

1. Зарегистрируйтесь на [openrouter.ai](https://openrouter.ai/)
2. Создайте API key
3. Создайте файл `config/secrets.local.env` и запишите туда:

```
OpenRouter_API_KEY=sk-or-v1-ВАШ_КЛЮЧ
```

Этот файл игнорируется в git и не попадёт в репозиторий.

**Модели:**

По умолчанию используется бесплатная модель `qwen/qwen3.6-plus-preview:free`.

Если хотите разрешить платные модели, добавьте в `.env`:

```bash
OPENROUTER_ALLOW_PAID=1
```

Более подробно — в `config/OPENROUTER.md`.

## Шаблон сопроводительного письма

1. Скопируйте шаблон:

```bash
cp config/cover-letter.example.txt config/cover-letter.txt
```

2. Отредактируйте `config/cover-letter.txt` — напишите, как вы обычно представляете себя: что важно упомянуть, какой стиль, какие формулировки.

Этот файл — **шаблон-затравка** для генерации писем. LLM будет использовать его как основу и адаптировать под конкретную вакансию.

**Эталоны стиля (опционально):**

Скопируйте примеры ваших реальных писем:

```bash
cp config/cover-letter-style-examples.example.txt config/cover-letter-style-examples.txt
```

Разделяйте несколько примеров строкой `---`. Это помогает LLM имитировать вашу манеру, а не генерировать «нейросетевой» текст.

## Сбор вакансий

```bash
npm run vacancies
```

**Что произойдёт:**

1. Откроется браузер с вашей сессией hh.ru
2. По очереди выполняются запросы из `config/search-keywords.txt`
3. Страницы вакансий сохраняются для дальнейшей обработки

```bash
npm run harvest
```

**Что произойдёт:**

1. Скрипт загружает собранные страницы вакансий
2. Для каждой вакансии отправляется запрос к OpenRouter с:
   - Текстом вакансии
   - Вашими резюме из `CV/`
3. Получает три оценки:
   - **scoreVacancy** — насколько вакансия релевантна
   - **scoreCvMatch** — насколько ваши навыки покрывают требования
   - **scoreOverall** — стоит ли откликаться
4. Результаты сохраняются в `data/vacancies-queue.json`

Оценки отфильтровывают очевидно неподходящие вакансии и помогают приоритезировать.

## Оценка через LLM

Скрипт `harvest` делает это автоматически. Но если хотите перезапустить оценку с новыми简历 или весами:

```bash
npm run harvest
```

**Веса оценок** настраиваются в `config/preferences.json`:

```json
{
  "llmScoreWeights": {
    "vacancy": 0.4,
    "cvMatch": 0.6
  }
}
```

Эти веса используются для расчёта `scoreOverall` как взвешенной суммы двух остальных.

## Работа с дашбордом

```bash
npm run dashboard
```

Откроется http://127.0.0.1:3849

### Интерфейс

Дашборд содержит несколько вкладок:

#### Vacancies — очередь ваканций

Здесь отображаются вакансии с оценками. Можно:

- Сортировать по любому полю
- Фильтровать по скорам
- Просматривать детали вакансии
- Удалять нерелевантные

#### Письма — генерация и редактирование сопроводительных

1. Выберите вакансию из списка
2. Нажмите «Сгенерировать» — LLM создаст письмо на основе:
   - Текста вакансии
   - Вашего шаблона (`config/cover-letter.txt`)
   - Ваших примеров стиля (если есть)
3. Отредактируйте письмо, если нужно
4. Нажмите «Утвердить» — письмо сохраняется в очередь

Утверждённые письма подмешиваются как эталоны при будущих генерациях.

#### Settings — настройки

Порт сервера, лимиты и прочие параметры.

## Отклик на вакансию

Есть два способа.

### Способ 1: Через форму отклика на странице вакансии (`hh-fill-letter`)

Открывает страницу, нажимает «Откликнуться», вставляет письмо и ждёт ручной проверки.

```bash
npm run hh-fill-letter -- --id=<uuid-записи-из-очереди>
```

uuid берётся из `data/vacancies-queue.json` (поле `id` у нужной записи).

Или по URL:

```bash
npm run hh-fill-letter -- --url=https://hh.ru/vacancy/123456 --text-file=./letter.txt
```

**Флаги:**

- `--stay-open` — не закрывать браузер (удобно для отладки)

### Способ 2: Через чат с работодателем (`hh-apply-chat`)

Письмо уходит **в переписку с работодателем**, а не в поле формы на карточке вакансии.

```bash
npm run hh-apply-chat -- --id=<uuid-записи-из-очереди>
```

Флаги:
- `--stay-open` — не закрывать браузер
- `--dry-run` — открыть чат, но не вставлять письмо
- `--no-submit` — открыть форму отклика, но не отправлять

### Способ 3: Из дашборда

На карточке вакансии в дашборде есть кнопка «Отклик в браузере» (доступна при утверждённом письме). Она запускает `hh-apply-chat` на той же машине, где запущен сервер дашборда.

## Обновление селекторов при смене вёрстки

Если при запуске скриптов появляются ошибки вроде «element not found» или selector не найден — hh.ru, возможно, изменил вёрстку.

1. Запустите генерацию селекторов:

```bash
npm run codegen-hh
```

2. Если не помогло, запустите вручную:

```bash
npx playwright codegen https://hh.ru
```

Откроется окно, в котором можно кликать по элементам и сразу получать селекторы. Скопируйте новые селекторы в:
- `lib/hh-response-selectors.mjs` — для формы отклика
- `lib/hh-chat-selectors.mjs` — для чата

## Решение типичных проблем

| Проблема | Решение |
|----------|---------|
| Скрипт не авторизуется на hh.ru | Запустите `npm run login` для обновления сессии |
| «Element not found» при отклике | Обновите селекторы через `npm run codegen-hh` или вручную |
| Скрипт зависает на открытии вакансии | Увеличьте `HH_OPEN_DELAY_MAX_MS` в `.env` |
| LLM даёт странные оценки | Проверьте CV в `CV/` — они должны быть читаемыми и подробными |
| Дашборд не запускается | Проверьте, свободен ли порт 3849, или задайте `DASHBOARD_PORT` |
| Письмо генерируется слишком длинным | Уменьшите `COVER_LETTER_STYLE_MAX_CHARS` в `.env` |
| Скрипт запускается, но ничего не делает | Проверьте `config/search-keywords.txt` — в нем должны быть запросы |

## Порядок действий для типичного сценария

Вот стандартный рабочий цикл:

1. **Один раз:** `npm run login` — войти в аккаунт
2. **Подготовка:** написать `config/search-keywords.txt`, резюме в `CV/`, шаблон письма в `config/cover-letter.txt`
3. **Сбор:** `npm run vacancies` — собрать вакансии
4. **Оценка:** `npm run harvest` — оценить через LLM
5. **Дашборд:** `npm run dashboard` — перейти в http://127.0.0.1:3849
6. **Письма:** для интересующих вакансий — «Сгенерировать» — проверить — «Утвердить»
7. **Отклик:** «Отклик в браузере» или `npm run hh-apply-chat -- --id=...`
8. Повторять шаги 3-7 по мере появления новых вакансий
