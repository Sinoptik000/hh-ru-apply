# UI Specification — hh-ru-apply Dashboard

## Обзор

Веб-интерфейс дашборда для управления вакансиями. Одностраничное приложение (SPA) на vanilla JS + CSS. Сервер: Express.js на порту 3849.

## Файлы

- `dashboard/public/index.html` — разметка
- `dashboard/public/app.js` — логика
- `dashboard/public/style.css` — стили

---

## Элементы UI

### Header (`.top`)

**Расположение:** верхняя часть страницы, фиксированная высота ~80px.

**Структура:**
```
┌─────────────────────────────────────────────────────────────────┐
│ [h1: "Очередь вакансий"]  [📝 Фильтры] [➕ Добавить] [🔍 Sourcing] [Лог отклика hh] │
└─────────────────────────────────────────────────────────────────┘
```

**Элементы:**
- `h1` — заголовок "Очередь вакансий", шрифт 1.35rem, font-weight 600
- `.btn-log-apply` — кнопка "Лог отклика hh", открывает модалку с логом
- `.btn-sourcing` — кнопка "🔍 Sourcing", запуск сбора вакансий
- `.btn-add-vacancy` — кнопка "➕ Добавить", добавляет вакансию из буфера обмена
- `.keywords-dropdown` — выпадающий список ключевых слов

**Sourcing прогресс-бар** (появляется при запуске sourcing):
- `.sourcing-progress-wrap` — контейнер, скрыт по умолчанию
- `.sourcing-progress-fill` — заполненная часть (зелёный градиент)
- `.sourcing-progress-text` — текст "0/0", обновляется в процессе

**Add vacancy прогресс-бар** (появляется при добавлении):
- `.add-vacancy-progress-wrap` — контейнер, скрыт по умолчанию
- `.add-vacancy-progress-fill` — заполненная часть
- `.add-vacancy-progress-text` — текст статуса

### Tabs (`.vacancy-tabs`)

**Расположение:** под header, над поиском.

**Кнопки:**
- `data-status="pending"` — "На проверке" (по умолчанию активна)
- `data-status="approved"` — "Подходят"
- `data-status="rejected"` — "Отклонённые"

**Поведение:**
- При клике переключает активный таб
- Внутри кнопки отображается счётчик вакансий: "На проверке (12)"
- Счётчики обновляются через `syncVacancyTabs()`

**Стили:**
- Активный таб: `color: var(--accent)`, `border-color: var(--accent)`
- Неактивный: `color: var(--muted)`, `border: 1px solid var(--border)`

---

### Tabs + Search Row (`.tabs-row`)

**Расположение:** под header, перед списком вакансий. Tabs и search bar на одном уровне, справа от tabs.

**Структура:**
```
[На проверке (12)] [Подходят (5)] [Отклонённые (3)]  [🔍 Поиск... ×]
```

**Стили:**
- Flex-контейнер с `gap: 0.75rem`, `flex-wrap: wrap`
- Высота строки ~40px, центрирование по вертикали
- `padding: 0 1.5rem 0.75rem` — отступы совпадают с карточками

### Search Bar (inline, `.search-bar-wrap`)

**Расположение:** справа от tabs в одном ряду.

**Элементы:**
- `#vacancy-search` — текстовое поле ввода, placeholder "Поиск..."
- `#vacancy-search-clear` — кнопка "×", скрыта по умолчанию, появляется при вводе текста

**Стили:**
- `flex: 1`, `min-width: 12rem`, `max-width: 24rem`
- Поле: padding 0.35rem 1.75rem 0.35rem 0.6rem, border-radius 6px
- Фокус: `border-color: var(--accent)`, тень `rgba(124, 183, 255, 0.15)`
- Кнопка очистки: позиция absolute справа 0.5rem

**Поведение:**
- При вводе текста вызывается `onSearchInput()` → `renderVacancyList()`
- Кнопка × вызывает `clearSearch()` → сбрасывает поиск
- Поиск сохраняется при переключении табов
- Фильтрация по: `title`, `company`, `geminiTags`, `searchQuery`

### Vacancy List (`#list`)

**Контейнер:** `<main id="list">`, padding 1rem 1.5rem 2rem, max-width 52rem.

**Карточка вакансии** (`.card`):

```
┌──────────────────────────────────────────────────────────────┐
│ [×]                                                           │
│  ┌──────┐  [Операционный директор (COO)]                      │
│  │  85  │  ООО Меридиан · от 120 000 до 250 000 ₽ · запрос:  │
│  └──────┘  COO                                               │
│                                                               │
│  Место работы: [Гибрид] [Офис]                               │
│  Английский: [B2]                                            │
│                                                               │
│  Ты подходишь на роль COO...                                 │
│  Нюансы: У тебя сильный операционный опыт...                 │
│                                                               │
│  [COO] [операционный директор] [управление проектами]         │
│                                                               │
│  [Отклик в браузере]                                         │
│                                                               │
│  [Черновик письма]  [Снова сгенерировать письмо]  [Письмо]   │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │ [↻]  [Создать сопроводительное]                         │ │
│  │                              [Подходит]  [Не подходит]   │ │
│  │  [textarea: Причина...]                                 │ │
│  └─────────────────────────────────────────────────────────┘ │
│                                                               │
│  Комментарий: ...                                            │
└──────────────────────────────────────────────────────────────┘
```

**Элементы карточки:**

| Элемент | Класс | Описание |
|---------|-------|----------|
| Кнопка закрытия | `.card-dismiss` | Удалить без обработки |
| Блок скоринга | `.score-block` | Число + тултип с разбивкой |
| Счёт | `.score` | Итоговый балл 0-100 |
| Тултип | `.score-tooltip` | Разбивка: вакансия, CV, итог |
| Инфо о модели | `.model-info-btn` | "i" — показывает модель OpenRouter |
| Ссылка на вакансию | `.title-link` | Название + ссылка на hh.ru |
| Мета-информация | `.meta` | Компания, зарплата, запрос |
| Место работы | `.workplace-badges` | Бейджи: удалёнка/гибрид/офис |
| Уровень языка | `.language-badge` | A1-C2, цветовая кодировка |
| AI-резюме | `.summary` | Текст `geminiSummary` |
| Риски | `.risks` | Текст `geminiRisks` |
| Теги | `.tags` | Список `geminiTags` |
| Кнопка отклика | `.btn-apply-chat` | "Отклик в браузере" |
| Черновик письма | `.cover-draft-btn` | Открывает модалку редактирования |
| Регенерация | `.btn-regenerate-letter` | Пересоздать письмо |
| Просмотр письма | `.btn-view-approved` | Открывает модалку с письмом |
| Экшены | `.actions` | Появляется для status=pending |
| Причина | `.done-reason` | Комментарий для отклонённых |

**Состояния карточки:**
- `status=pending` — показаны кнопки "Создать сопроводительное", "Подходит", "Не подходит"
- `status=approved` — показаны "Отклик в браузере", "Письмо"
- `status=rejected` — показан `.done-reason` с комментарием

**Цветовая кодировка скора** (через background):
- score > 75 — зелёный
- score 50-75 — жёлтый
- score < 50 — красный

### Модалки

#### Draft Modal (`#draft-modal`)

Для редактирования черновика сопроводительного письма.

**Структура:**
- Заголовок: "Черновик письма"
- Название вакансии
- Fieldset с радио-кнопками вариантов (1, 2, 3)
- Textarea для редактирования
- Кнопки: "Сохранить правки", "Утвердить", "Отклонить"

#### Apply Log Modal (`#apply-log-modal`)

Для просмотра лога отклика в браузере.

**Структура:**
- Заголовок: "Лог «Отклик в браузере»"
- Путь к файлу лога
- Кнопка "Обновить"
- `<pre>` с текстом лога

#### Approved Letter Modal (`#approved-letter-modal`)

Для просмотра утверждённого письма.

**Структура:**
- Заголовок: "Утверждённое письмо"
- Название вакансии
- `<pre>` с текстом письма
- Кнопка "Копировать"

## Состояния (Empty/Error)

| Состояние | Элемент | Текст |
|-----------|---------|-------|
| Пустой список | `.empty` | "Пусто." |

## Адаптивность

- `max-width: 52rem` для основного контента
- `max-width: min(42rem, 100%)` для модалок
- Flex-wrap для кнопок и табов
- Карточки адаптируются по ширине через flex/grids
