# Главная страница — Очередь вакансий

## Описание

Главная страница дашборда с списком вакансий, разбитым по статусам через вкладки.

**Файлы:**
- `dashboard/public/index.html`
- `dashboard/public/app.js`
- `dashboard/public/style.css`

## Вкладки (Tabs)

| Вкладка | data-status | Описание |
|---------|-------------|----------|
| Вручную | `manual` | Вакансии добавленные вручную |
| На проверке | `pending` | Требуют оценки |
| Подходят | `approved` | Одобреные |
| Отклонённые | `rejected` | Отклонённые |

**Счётчики** обновляются через `syncVacancyTabs()`.

**Активный таб:** `color: var(--accent)`, `border-color: var(--accent)`

## Кнопки "Обновить"

### 1. Progress Ring — в header секции (#btn-refresh-section)

Расположена в header дашборда справа от поля поиска. Круглая кнопка 72×72px с кольцевым прогресс-баром.

**UI-состояния:**

| Состояние | Визуал |
|-----------|--------|
| Default | Иконка 🔄 + подпись "N вакансий" под кнопкой |
| Loading | Вращающееся кольцо (::before) + пульсирующая иконка + подпись "0 / N" |
| Готово | Кольцо заполнено на 100%, сбрасывается через 600ms |

**Логика видимости:**
- Показывается только когда в текущем разделе ≥1 вакансия с `descriptionPreview < 80` символов
- Скрывается: нет данных, нет неполных, уже идёт обновление

**Поведение при клике:**
1. `setRefreshSectionLoading(true, batch.length)` — блокирует, Progress Ring = 0%, подпись "0 / N"
2. До **20 вакансий** из текущей вкладки фильтруются по `needsVacancyBodyRefresh`
3. Каждая: `POST /api/vacancy/refresh-body` → пересчёт score
4. После каждой: `updateRefreshSectionProgress()` — кольцо заполняется, подпись "K / N"
5. `invalidateCache()` → `load()` для перерисовки
6. Toast: "Обновлено K из N. Ещё M — нажмите ещё раз"

**Технология:** Playwright с persistent-профилем (`data/session/chromium-profile/`), headless=true, networkidle, retry-парсинг.

---

### 2. В карточке вакансии (.btn-refresh-vacancy)

Кнопка-иконка (🔄) внутри карточки. Обновляет **конкретную одну** вакансию.

- CSS: `.btn-icon.btn-refresh-vacancy`
- API: `POST /api/vacancy/refresh-body` с `{ id }`
- После обновления: пересчёт score, перерисовка карточки
- Текст подсказки: "Обновить текст вакансии с hh.ru и заново оценить совпадение с CV (OpenRouter)"

---

### Критерий неполной вакансии

```javascript
needsVacancyBodyRefresh(item) {
  const desc = item.descriptionPreview || '';
  return desc.trim().length < 80;
}
```

### API refresh-body

| Эндпоинт | Метод | Тело | Описание |
|----------|-------|------|----------|
| `/api/vacancy/refresh-body` | POST | `{ id: string }` | Загружает текст с hh.ru и пересчитывает score через OpenRouter |
