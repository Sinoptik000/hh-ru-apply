# Вкладка "Вручную"

## Описание

Страница для ручного добавления вакансий по URL и работы с сопроводительными письмами.

**Файл:** `dashboard/public/app.js` (функции `renderManualTabUI`, `populateManualVacancySection`, `initManualTabHandlers`)

---

## Состояния

| Состояние | UI |
|-----------|-----|
| **Пустое** | Форма с input для URL, placeholder "Вставьте ссылку..." |
| **Загрузка** | Input disabled, кнопка "Загрузка..." |
| **Заполнено** | Все блоки с данными вакансии |
| **Ошибка** | Красное сообщение под input |

---

## Блоки (в порядке отображения)

### Блок 0: Ввод URL

```
┌─────────────────────────────────────────────────────────┐
│ Ссылка на вакансию hh.ru                               │
│ [https://hh.ru/vacancy/12345678         ] [Загрузить]   │
└─────────────────────────────────────────────────────────┘
```

**Элементы:**
- `.manual-url-input` — input type="url"
- `.btn-load-vacancy` — кнопка "Загрузить" / "Загрузка…"

**Валидация:** regex `^https?:\/\/([^.]+\.)?hh\.ru\/vacancy\/\d+`

---

### Блок 1: Заголовок + Скор

```
┌─────────────────────────────────────────────────────────┐
│ Вручную                                                [85] i│
│ Операционный директор (COO)                              │
└─────────────────────────────────────────────────────────┘
```

**Позиция:** Скор в правом верхнем углу

**Элементы:**
- `.manual-page-title` — "Вручную" (верхний левый угол)
- `.manual-vacancy-title` — название вакансии
- `.manual-score-block` — блок со скором (как на карточках)
  - `.score` — число 0-100
  - `.score-tooltip` — тултип с разбивкой
  - `.model-info-btn` / `.model-info-panel` — модель

**Цветовая кодировка скора:**
- > 75 — зелёный
- 50-75 — жёлтый
- < 50 — красный

### Блок 3: Описание вакансии

```
Описание вакансии
─────────────────────────────────────────────────────────
[Триггерная фраза если есть]

Текст описания с форматированием:
- Абзацы с пустыми строками между ними
- Списки для разделов "Требования" и "Задачи"
- Маркированные и нумерованные списки
─────────────────────────────────────────────────────────
```

**Форматирование:**
- Обычный текст → `<p>` с пустой строкой между
- Строки начинающиеся с `- `, `• `, `* ` → `<li>` в `<ul>`
- Строки с `1.` `2)` → `<li>` в `<ol>`
- Заголовки разделов "требования", "задачи", "обязанности" → `<p><strong>`

### Блок 4: Выбор варианта письма

```
Вариант
○ Вариант 1   ○ Вариант 2   ○ Вариант 3
```

**Элементы:**
- `fieldset` с 3 радио-кнопками
- Название "Вариант" в legend

---

### Блок 5: Редактирование письма

```
Текст (правки сохраняются и учитываются при следующей генерации)
┌─────────────────────────────────────────────────────────────┐
│ [textarea rows=12 с текстом выбранного варианта]            │
│                                                             │
└─────────────────────────────────────────────────────────────┘

[Сохранить правки]  [Утвердить]  [Отклонить]
```

**Кнопки:**
| Кнопка | Действие | API |
|--------|----------|-----|
| Сохранить правки | Сохраняет все варианты | `POST /api/cover-letter/save-draft` |
| Утвердить | Сохраняет текст как approvedText, переходит в "Подходят" | `POST /api/cover-letter/action` action=approve |
| Отклонить | Показывает поле причины | `POST /api/action` action=reject |

**Поле причины** (появляется при нажатии "Отклонить"):
```
Причина отклонения
┌─────────────────────────────────────────────────────────────┐
│ [textarea placeholder="Укажите причину..."]                │
└─────────────────────────────────────────────────────────────┘
[Подтвердить отклонение]  [Отмена]
```

---

### Блок 6: Финальные действия

```
[Сгенерировать письмо]  [Очистить]
```

**Кнопки:**
| Кнопка | Действие |
|--------|----------|
| Сгенерировать письмо | Запускает повторную генерацию вариантов |
| Очистить | Удаляет вакансию, сбрасывает форму (БЕЗ подтверждения) |

---

## Действия и результаты

| Действие | Кнопка | Результат |
|----------|--------|-----------|
| Загрузить вакансию | "Загрузить" | Парсит hh.ru, показывает данные, скор |
| Сгенерировать письмо | "Сгенерировать письмо" | Создаёт 3 варианта |
| Переключить вариант | Радио "Вариант 1/2/3" | Меняет текст в textarea |
| Редактировать | Textarea | Можно править текст |
| Сохранить правки | "Сохранить правки" | Сохраняет варианты на сервер |
| Утвердить | "Утвердить" | Текст → approved, вакансия в "Подходят" |
| Отклонить | "Отклонить" | Показывает поле причины |
| Подтвердить отклонение | "Подтвердить отклонение" | `action=reject`, форма очищается |
| Отмена | "Отмена" | Скрывает поле причины |
| Очистить | "Очистить" | Удаляет вакансию, сбрасывает форму |

---

## Flow: Утверждение вакансии

```
1. Вводим URL → "Загрузить"
2. Парсинг → показывает данные + скор
3. "Сгенерировать письмо" → 3 варианта
4. Выбираем вариант (радио) → правим в textarea
5. "Утвердить" → вакансия в "Подходит", переход на вкладку
```

---

## Flow: Отклонение вакансии

```
1. "Отклонить" → появляется поле причины
2. Вводим причину
3. "Подтвердить отклонение" → vacancy.status = rejected
4. Форма очищается
```

---

## Flow: Очистка

```
1. "Очистить" → без подтверждения
2. DELETE /api/dismiss с vacancyId
3. Форма сбрасывается
```
