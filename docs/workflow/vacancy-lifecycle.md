# Жизненный цикл вакансии

## Обзор

Вакансия проходит путь от сбора до автоматического отклика через браузер.

```
Сбор → Очередь → Скоринг → Решение → Отклик
```

---

## Стадии

### 1. Сбор (Sourcing)

**Как:**
- `npm run vacancies` — парсинг по ключевым словам из `config/search-keywords.txt`
- `npm run scan-tg` — сканирование Telegram каналов
- `➕ Добавить` в дашборде — вставка URL из буфера

**Результат:**
- Вакансия создаётся в статусе `manual` (добавлена вручную) или `pending` (автоматически)
- Заполняются: url, title, company, salary, description
- `status = "manual"` или `status = "pending"`

**Файл данных:** `data/vacancies-queue.json`

---

### 2. Скоринг (Harvest)

**Как:** `npm run harvest`

**Что делает:**
1. Для каждой вакансии без score:
   - Получает текст с hh.ru (description)
   - Отправляет в AI cascade (OpenRouter → Google → Groq)
   - Получает scores: vacancy, cvMatch, overall
   - Получает summary, risks, tags

**Результат:**
- `scoreVacancy` — оценка модели
- `scoreCvMatch` — сходство с CV
- `scoreOverall` — итоговый (взвешенный)
- `geminiSummary` — AI-резюме
- `geminiRisks` — нюансы
- `geminiTags` — теги
- `openRouterModel` — какая модель использовалась

---

### 3. Ручная проверка (Dashboard)

**Вкладки:**
- `На проверке` (pending) — новые вакансии
- `Подходят` (approved) — одобренные
- `Отклонённые` (rejected) — отклонённые

**Действия на карточке:**

| Действие | Кнопка | Результат |
|----------|--------|-----------|
| Создать письмо | "Создать сопроводительное" | Генерирует варианты |
| Подходит | "Подходит" | status → approved |
| Не подходит | "Не подходит" | status → rejected |

**С письмом:**
- Черновик письма → редактирование → Утвердить/Отклонить
- "Отклик в браузере" — запуск автоматизации

---

### 4. Решение по письму

**流程:**
1. Загрузить/добавить вакансию (скрапинг hh.ru)
2. Автоматически запускается генерация письма
3. При необходимости нажать "Сгенерировать письмо" вручную в блоке "Вариант"
4. Открыть "Черновик письма" или работать во вкладке "Вручную"
5. Выбрать вариант (радио) → отредактировать
6. "Утвердить" → approvedText сохранён

**Статусы coverLetter:**
- `pending` — есть варианты, не утверждено
- `approved` — есть approvedText
- `declined` — отклонено

---

### 5. Отклик (Apply)

**Как:** кнопка "Отклик в браузере" на карточке

**Что делает:**
1. Открывает Chromium с сохранённым профилем
2. Переходит на hh.ru/vacancy/{id}
3. Заполняет форму отклика
4. Вставляет approvedText
5. Отправляет

**Лог:** `data/hh-apply-chat.log`

---

## Статусы в данных

```javascript
{
  status: "pending" | "approved" | "rejected" | "manual",
  coverLetter: {
    status: "pending" | "approved" | "declined",
    variants: string[],      // 3 варианта
    approvedText: string     // утверждённый текст
  }
}
```

## Файлы данных

| Файл | Назначение |
|------|------------|
| `data/vacancies-queue.json` | Основная очередь вакансий |
| `data/feedback.jsonl` | История approve/reject |
| `data/skipped-vacancies.jsonl` | Отклонённые фильтрами |
| `data/rejected-vacancy-ids.jsonl` | ID отклонённых |
| `data/hh-apply-chat.log` | Лог откликов |
